## 一、实习背景与目标

### 1.1 实习背景

本次实习以 **网络安全攻防** 为核心方向，通过构建一个功能完整的 Web 应用，并在开发过程中**主动引入、发现、修复各类安全漏洞**，深入理解常见 Web 漏洞的原理、攻击手法与防御方案。

采用 **"先构建、再攻破、后修复"** 的实战化学习模式，覆盖了 OWASP Top 10 中的大部分漏洞类型。

### 1.2 实习目标

| 目标 | 说明 |
|------|------|
| 🎯 掌握 Flask Web 开发基础 | 路由、模板、表单处理、Session 管理 |
| 🎯 理解常见 Web 漏洞原理 | SQL 注入、XSS、CSRF、SSRF、文件上传、命令注入、XXE 等 |
| 🎯 实战漏洞挖掘与利用 | 手动构造攻击载荷验证漏洞 |
| 🎯 掌握安全修复方案 | 参数化查询、输入校验、白名单、CSRF Token 等 |
| 🎯 撰写安全报告 | 每类漏洞输出详细的测试与修复报告 |

## 二、项目概述

### 2.1 项目简介

用户信息管理平台是一个基于 **Python Flask + SQLite** 构建的 Web 应用，提供用户登录、注册、搜索、头像上传、个人中心、余额充值、密码修改、URL 抓取、Ping 网络诊断、XML 数据导入等功能。项目代码托管于 GitHub，累计 **17 次提交**，包含 **9 个模板文件**、**459 行核心代码**。

### 2.2 技术栈

| 技术 | 用途 |
|------|------|
| **Python 3 / Flask** | Web 框架 |
| **SQLite3** | 用户注册数据存储 |
| **Werkzeug** | 密码哈希与安全工具 |
| **HTML5 + CSS3** | 前端页面与样式 |
| **Git / GitHub** | 版本控制与代码托管 |
| **systemd** | Linux 服务管理 |
| **tmux** | 终端会话管理 |

### 4.2 安全思维转变

| 转变 | 说明 |
|------|------|
| 🧠 **从"能用就行"到"安全第一"** | 开发时开始思考"这个接口会不会被攻击" |
| 🧠 **从"被动防御"到"主动测试"** | 学会像攻击者一样思考，主动构造恶意载荷测试 |
| 🧠 **从"单一防护"到"纵深防御"** | 理解多层防御架构，不依赖单一点的安全措施 |

### 4.3 对漏洞的理解变化

```
实习前："SQL 注入就是在输入框里加个引号"
实习后：理解了 SQL 注入 → 闭合 + 联合查询 + 盲注 + 时间注入 + 绕过防御

实习前："文件上传漏洞就是上传 PHP 文件"
实习后：理解了文件上传 → 扩展名绕过 → MIME 伪造 → 图片马 → 路径遍历 → Magic Bytes 校验绕过

实习前："CSRF 不太懂"
实习后：理解了 CSRF → Token 同步模式 → SameSite Cookie → Referer 校验 → 防御架构
```

## 五、遇到的问题与解决方案

### 5.1 技术问题

| 问题 | 解决方案 |
|------|----------|
| Flask 端口被占用 | 使用 `lsof -t -i:端口` 查找并清理 |
| CSRF Token 校验与表单同步 | 使用 `context_processor` 全局注入 Token |
| frp 公网服务器需要备案 | 改用花生壳内网穿透 + serveo.net 隧道备份 |
| Git push 超时（网络拦截） | 使用 Token 认证 + 重试机制 |
| tmux 开机自启进程退出 | 使用 `bash -c "claude; exec bash"` 保持会话 |
| XXE 解析器默认行为 | 四层正则过滤 + 清空 entity 映射 |

### 5.2 安全修复中的权衡

| 场景 | 权衡 | 最终方案 |
|------|------|----------|
| URL 抓取功能的便利 vs 安全 | 完全开放 → 限制作为妥协 | 协议白名单 + IP 过滤，保留核心功能 |
| Ping 功能需要 shell 执行 | shell=True 方便 → shell=False 安全 | 参数列表模式，保留功能但禁用 Shell |
| XML 导入需解析实体 | 支持 ENTITY → 禁止 ENTITY | 移除所有 DTD 定义，只解析纯 XML |

## 六、安全加固总结

### 6.1 安全防护体系

经过整个实习期的安全加固，项目形成了 **五层安全防护体系**：

```
第一层：输入校验层
├── 参数化查询（防 SQL 注入）
├── 正则白名单（防命令注入）
├── HTML 自动转义（防 XSS）
└── DTD 过滤（防 XXE）

第二层：身份认证层
├── 密码哈希存储（PBKDF2+sha256）
├── Session 会话管理
└── 关键操作需登录

第三层：权限控制层
├── Session user_id 强制校验（防越权）
├── CSRF Token 校验（防跨站请求）
└── 白名单页面访问（防文件包含）

第四层：数据安全层
├── 密码不外传前端
├── 余额整数分存储（防精度丢失）
└── 控制台日志脱敏

第五层：文件安全层
├── 扩展名白名单校验
├── MIME 类型校验
├── Magic Bytes 文件签名验证
├── UUID 重命名防覆盖
└── 路径遍历防护（basename + realpath）
```

## 七、实习心得与体会

### 7.1 对网络安全的认识

通过这次实习，我深刻认识到 **"没有绝对的安全"**。每一个看似简单的功能背后，都可能隐藏着意想不到的攻击面。一个不加校验的文件上传、一个不设防的输入框、一个缺少 Token 的表单，都可能成为整个系统的突破口。

### 7.2 开发与安全的平衡

安全与便利之间总是存在矛盾。过于严格的安全措施会影响用户体验，过于宽松则会导致漏洞频发。通过这次实习，我学会了如何在两者之间找到平衡点：**在不影响核心功能的前提下，实施最低权限、纵深防御的安全策略**。

### 7.3 核心收获

> **"如果你不知道如何攻击一个系统，你就不知道如何防御它。"**

这次实习让我真正理解了这句话的含义。通过亲手构造攻击载荷、亲眼看到漏洞被利用、亲手修复并验证，我对 Web 安全的理解从书本上的理论变成了实实在在的实践经验。

感谢实习单位提供的学习平台和指导，感谢导师在项目开发和安全审计中的悉心指导。这次实习经历将为我未来在网络安全方向的学习和工作奠定坚实的基础。
